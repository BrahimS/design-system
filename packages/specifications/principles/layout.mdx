### Spacing concepts

On top of spacing baseline, we have a few concepts that are useful to work with spacing.
Here are five spacing concepts described afterwards to apply to every ODS component:

#### Stack

The stack concept is used for vertical spacing between two elements (like carriage return).

## Grid
_Every layout should sit on a grid that brings order and hierarchy to the interface. Define a grid separately for mobile, tablet and desktop devices with columns, gutters, and margins so your interface can adapt to any platform easily._

## Breakpoints

| name                  | purpose                                                           | size(rem) | size(px)   |
|-- --------------      | ----------------------------------------------------------------- | --------  | ---------- |
| `--ods-breakpoint-xs` | For small viewports as mobile devices (in portrait)               | 36rem     | 576px      |
| `--ods-breakpoint-sm` | For smartphones (in landscape) or tablet (in portrait) viewports  | 48rem     | 768px      |
| `--ods-breakpoint-md` | For small desktops or tablets (in landscape) viewports            | 62rem     | 992px      |
| `--ods-breakpoint-lg` | For desktops viewports                                            | 75rem     | 1200px     |
| `--ods-breakpoint-xl` | For widescreen desktops                                           | 87.5rem   | 1400px     |
